# Support

DesktopLab uses separate GitHub channels so public questions, actionable defects and confidential security reports do not become mixed together.

## Get Help

Use [GitHub Discussions Q&A](https://github.com/Vitalisimon/desktoplab/discussions/categories/q-a) for installation help, hardware compatibility, runtime and model configuration, and general questions.

Search existing discussions and the [troubleshooting guide](docs-public/troubleshooting.md) before starting a new discussion.

## Report A Bug

Use the structured [bug report form](https://github.com/Vitalisimon/desktoplab/issues/new/choose) for a reproducible DesktopLab defect. Include the exact version, package type, operating system and smallest reproduction sequence.

Review diagnostic bundles before attaching them. Never publish credentials, private paths, repository content, prompts or raw tool output.

## Request A Feature

Use the [feature request form](https://github.com/Vitalisimon/desktoplab/issues/new/choose) for a concrete user problem and desired outcome. Open-ended ideas and early product discussion belong in GitHub Discussions.

## Report A Vulnerability

Do not open a public issue. Follow [SECURITY.md](SECURITY.md) and use the repository's active GitHub Private Vulnerability Reporting channel.

## Support Scope

Support claims are limited to the platforms, runtimes and packages listed in [platform support](docs-public/platform-support.md) and [runtime and provider support](docs-public/runtime-and-provider-support.md). Development evidence does not create a public support commitment.
